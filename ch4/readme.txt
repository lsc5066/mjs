-변수는 값을 저장하기 위해 확보한 메모리 공간 자체를 의미한다. 이를 식별하기 위해 이름을 붙인 것.
-식별자(=변수 이름)은 어떤 값을 구별해서 식별할 수 있는 고유한 이름을 의미함.
-변수 선언은 런타임 이전에 먼저 실행, 이후 변수 할당은 런타임에 실행됨.(본인의 소스코드 위치 순서대로)
-변수 재할당은 이전 값을 저장하고 있던 메모리 공간을 지우고 새로운 값을 저장하는 것이 아닌, 새로운 메모리 공간을 확보하고 그 공간에 새로운 값을 저장하는 개념.
-식별자와 연결되어 있지 않는 불필요한 값들은 가비지 콜렉터에 의해 메모리에서 자동으로 해제됨.
-변수나 함수의 이름에는 카멜 케이스를 사용(camelCase)하고 생성자 함수, 클래스의 이름에는 파스칼 케이스(PascalCase)를 사용함.
